# Amazon Simple Storage Service (Amazon S3)

## Overview

Amazon Simple Storage Service (Amazon S3) is an object storage service that provides secure, durable, and highly available storage. In the DataShield platform, Amazon S3 is used to store processed files generated by the Analyzer Service before triggering further processing through AWS Lambda.

---

# Purpose in DataShield

Amazon S3 serves as the centralized storage layer for processed data.

It is responsible for:

- Storing processed JSON files
- Providing durable and highly available storage
- Triggering AWS Lambda automatically
- Decoupling the Analyzer Service from the database layer
- Maintaining processed files for future retrieval and auditing

---

# Why Amazon S3?

Amazon S3 was selected because it provides:

- 99.999999999% (11 nines) durability
- High availability
- Unlimited storage scalability
- Event-driven architecture through S3 Event Notifications
- Secure IAM integration
- Cost-effective object storage

# Bucket Configuration

| Property | Value |
|-----------|-------|
| Bucket Type | General Purpose |
| Region | ap-south-1 |
| Versioning | Enabled |
| Public Access | Blocked |
| Encryption | SSE-S3 (or your configured encryption) |
| Object Ownership | Bucket Owner Enforced |

# Files Stored

The bucket stores:

- Processed JSON files
- Metadata files
- Output generated by the Analyzer

Example:

```
processed/
    result_001.json
    result_002.json
    result_003.json
```

---

# Event Notification

Whenever a new object is uploaded:

Amazon S3 automatically triggers the Lambda function.

```
Analyzer

↓

Upload JSON

↓

Amazon S3

↓

Object Created Event

↓

Lambda Triggered

↓

Insert Metadata into Amazon RDS
```

---

# IAM Integration

The Analyzer EC2 instance uploads files using an IAM Role.

No AWS Access Keys are stored inside the application.

Permissions are provided through:

- IAM Role
- Instance Profile

This improves security by using temporary credentials.

---

# Security Features

- Public Access Block enabled
- Bucket accessible only through IAM Roles
- No public objects
- Secure communication over HTTPS
- Bucket Owner Enforced
- Encryption at Rest

# Challenges Faced

During development, the following issues were encountered:

- IAM permission errors while uploading objects
- Lambda trigger configuration
- Bucket policy verification
- Object path validation
- Metadata extraction testing

These issues were resolved by configuring IAM Roles correctly and validating the S3 Event Notification configuration.

---

# Best Practices Followed

- Public access completely blocked
- Least Privilege IAM permissions
- Versioning enabled
- Bucket encryption enabled
- Event-driven architecture
- No AWS credentials stored in application code

---

# Key Takeaways

Amazon S3 acts as the central storage component of the DataShield platform. It provides secure and durable storage for processed files while enabling an event-driven workflow by automatically triggering AWS Lambda whenever new objects are uploaded. This design improves scalability, security, and separation of responsibilities between application components.
